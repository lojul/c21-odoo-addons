================================================================================
CASE STUDY: AI-Powered Property Listing Automation
C21NET Real Estate Platform
================================================================================

PROJECT OVERVIEW
--------------------------------------------------------------------------------
Built an end-to-end automated system for processing property listing PDFs,
extracting data using AI, and publishing to a real estate website with an
AI-powered chatbot for customer inquiries.

TECHNOLOGY STACK
--------------------------------------------------------------------------------
- Odoo 19 (ERP/CRM)
- n8n (Workflow Automation)
- Azure AI Services (Document Intelligence, AI Search, Blob Storage)
- OpenRouter API (LLM - Llama 3.1 70B)
- Microsoft OneDrive (File Storage)
- PDF.co (PDF to Image Conversion)
- Vercel (Website/API Hosting)
- Digital Ocean (Odoo Hosting with Docker + Caddy)

================================================================================
SYSTEM ARCHITECTURE
================================================================================

PDF Import Flow:
----------------
1. User uploads PDF via Odoo → n8n webhook → OneDrive
2. OneDrive Trigger detects new file → Starts processing workflow
3. Azure Document Intelligence extracts text and structure
4. AI (Llama 3.3) extracts structured property data (name, address, rent, etc.)
5. Data indexed to Azure AI Search (vector embeddings for RAG)
6. Property listing created/updated in Odoo
7. PDF pages converted to images via PDF.co → Uploaded to Odoo

Website Integration:
--------------------
Odoo (Property Data) → Vercel API → React Website
         ↓
Public Image Controller (/property/image/)
         ↓
AI Chatbot (RAG with Azure AI Search + Llama)

N8N WORKFLOWS DEVELOPED
================================================================================

1. Property PDF Processor v0.7 (Main Workflow)
----------------------------------------------
   Nodes: OneDrive Trigger → Config → Filter PDFs → Download PDF →
          Process One by One → Prepare Upload → Upload to Azure Blob →
          Submit OCR → Poll OCR Result → Extract Text & Structure →
          Extract & Describe Images → Prepare Extraction Prompt →
          AI Property Extractor (Llama 3.3) → Parse Property JSON →
          Chunk, Embed & Index → Create/Update Odoo Listing →
          Upload Images to Odoo

   Key Features:
   - Deduplication by source filename
   - Automatic property type/business type lookup in Odoo
   - One-page-at-a-time PDF conversion (handles PDF.co 0-based indexing)
   - URL encoding for Chinese filenames in Azure Blob
   - Graceful end-of-PDF detection via 454 error handling

2. Odoo Upload to OneDrive (Helper Workflow)
--------------------------------------------
   Nodes: Webhook → Prepare File → Upload to OneDrive → Respond

   Purpose: Allows Odoo to upload PDFs directly to OneDrive, which then
   triggers the main processing workflow automatically.

================================================================================
KEY TECHNICAL CHALLENGES SOLVED
================================================================================

1. Chinese Filename URL Encoding
   - Problem: PDF.co API returned 454 error for Chinese filenames
   - Solution: URL-encode blob path segments while preserving slashes
   - Code: data.blobPath.split("/").map(p => encodeURIComponent(p)).join("/")

2. PDF.co 0-Based Page Indexing (Undocumented)
   - Problem: PDF.co returned "page out of range" errors even for valid pages
   - Discovery: PDF.co uses 0-based indexing (page 0 = first page) despite
     documentation suggesting 1-based indexing
   - Solution: Convert pages one at a time starting from index 0, stop when
     454 error indicates end of PDF
   - Code: for (let pageIndex = 0; ...) { pages: String(pageIndex) }

3. PDF.co /pdf/info Response Structure
   - Problem: Page count not accessible at expected location
   - Discovery: PageCount is nested in info.PageCount (PascalCase), not pageCount
   - Solution: Access via infoResponse.info.PageCount

4. Odoo Field Type Mapping
   - Problem: n8n code used wrong field names (property_type vs property_type_id)
   - Solution: Look up Many2one IDs from Odoo, use Many2many command format
   - Code: [[6, 0, [ids]]] for Many2many fields

5. Webhook Integration Without Modifying Main Workflow
   - Problem: Adding Odoo trigger broke existing OneDrive workflow
   - Solution: Separate upload-to-OneDrive workflow that feeds into main workflow

6. Status Callback from n8n to Odoo
   - Problem: Odoo import status stuck on "Processing"
   - Solution: Embed log ID in filename (_ODOO123.pdf), parse at workflow end,
     call Odoo callback API endpoint

7. Vector Index Cleanup
   - Problem: Need to delete indexed chunks when re-importing
   - Solution: Azure AI Search REST API for document deletion by ID
